The code below can be copied directly into the Arduino sketch editor.

You will need the ClockBuilder Pro app from here: https://www.skyworksinc.com/Application-Pages/Clockbuilder-Pro-Software

It's not my intention to give instructions on the use of this software as it's beyond the scope of these instructions.

Using Clockbuilder Pro you can specify the frequencies you desire and it will output various text files containing the register information required, and I have put samples of such
in the Clockbuilder_Samples directory of this repository.

The registry values can then be inserted into the //Clockbuilder Pro Register dump section, replacing what is in the section below.

NOTE: The value pairs have a decimal address and a hex value in the sketch below. You may need to manipulate/edit the Clockbuilder Pro output to ensure it has the same format as below.

Co-Pilot is very good at converting this for you.

You will need an appropriate library for your ATtiny85 in order to export to a suitable hex file. I use the Digistump/Digispark library.

Once your sketch is edited to your requirements, use 'SKETCH/Export compiled Binary' which will then create a hex file that can be programmed to the ATtiny85 using a TL866 or similar.

This is the way I do it, you can do it in whatever way works for you.


CODE ONLY BELOW:

void writeReg(uint8_t reg, uint8_t val) {
    TinyWireM.beginTransmission(SI5351_ADDR);
    TinyWireM.write(reg);
    TinyWireM.write(val);
    TinyWireM.endTransmission();
}

void setup() {

    TinyWireM.begin();

    // Write all registers
    for (uint8_t i = 0; i < sizeof(regs)/2; i++) {
        writeReg(regs[i][0], regs[i][1]);
    }
    // Optional: reset PLLs (recommended by Silicon Labs)
    // Register 177: bit 7 = reset PLLA, bit 6 = reset PLLB
    // writeReg(177, 0xA0);

    // Done
}

void loop() {
    // Nothing to do but flash the LED to say done â€” ATtiny just configures the Si5351 once at boot

    delay(100);
    digitalWrite(LED_PIN, HIGH);  // LED OFF
    delay(100);
}
